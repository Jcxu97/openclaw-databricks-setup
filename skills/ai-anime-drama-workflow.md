# 第零章：Prompt 工程基础

> 好的 Prompt 是整个工作流的基石。以下原则贯穿所有环节。

## 0.1 六大常见 Prompt 错误与修正

### 错误一：描述模糊

- **错误示例**：`一个好看的女孩站在街上`
- **问题**：AI 无法判断"好看"的标准、场景风格、光线氛围
- **修正**：使用**具体客观描述**
  ```
  一位18岁东亚女孩，黑色长发及腰，穿白色连衣裙，
  站在东京涩谷十字路口，黄昏逆光，暖色调
  ```

### 错误二：缺少风格锚定

- **错误示例**：只描述内容，不指定风格
- **修正**：加入风格关键词
  ```
  anime style, cel shading, Studio Ghibli aesthetic,
  soft lighting, pastel color palette
  ```

### 错误三：一次性塞入过多内容

- **错误示例**：一个 prompt 试图描述整个场景 + 人物 + 动作 + 情绪 + 背景
- **修正**：分层描述，控制在 3-5 个核心要素
  - 主体 → 动作 → 环境 → 光线 → 风格

### 错误四：忽略负面提示词 (Negative Prompt)

- **修正**：常见负面提示词
  ```
  low quality, blurry, deformed hands, extra fingers,
  bad anatomy, watermark, text, signature
  ```

### 错误五：使用主观形容词

- **错误示例**：`beautiful`, `cute`, `cool`, `epic`
- **修正**：替换为客观描述
  - `beautiful` → `symmetrical face, high cheekbones, almond-shaped eyes`
  - `cool outfit` → `black leather jacket, silver chain necklace, dark sunglasses`

### 错误六：不区分模型的 Prompt 语法

- **Flux/SDXL**：自然语言描述，逗号分隔
- **SD 1.5**：权重语法 `(keyword:1.2)`, `[keyword]`
- **Midjourney**：支持 `--ar`, `--cref`, `--style` 等参数
- **集梦**：中文描述，支持 `@` 引用参考图

## 0.2 Prompt 工程核心原则

### 原则一：具体性 (Specificity)

- 不说"一个场景"，说"一个下雨天的东京街头，霓虹灯倒映在积水中"
- 不说"好看"，说"4K, photorealistic, cinematic lighting"

### 原则二：角色扮演 (Role-Playing)

- 让 AI 扮演专业角色可以显著提升输出质量
  ```
  你是一个专业的动画编剧，拥有10年的日本动画行业经验...
  你是一个专业的摄影师，擅长电影级构图...
  ```

### 原则三：迭代优化 (Iteration)

- 不要期望一次出完美结果
- 第一次：生成基础版本
- 第二次：基于结果调整细节
- 第三次：微调风格和构图

### 原则四：结构化格式

- 使用分隔符划分不同部分：`---`, `###`, `【】`
- 使用编号列表让 AI 理解优先级
- 使用 Markdown 格式增强可读性

## 0.3 六维 Prompt 框架（Seedance 2.0 / 通用）

> 来自 Seedance 2.0 社区 Prompt 库的最佳实践。六个维度，无需全部填写，缺省维度由模型自动补全。

### 维度一：输入 (Input)

用 `@` 符号引用参考素材：

```
@Image1 as first frame, @Video1 for camera movement,
@Video2 for character motion, @Audio1 for background music
```

### 维度二：内容 (Content)

描述**正在发生什么**——故事本身，不含运镜或时间：

```
[风格]. @[素材] [角色] 在 [环境] 中 [动作].
[角色] 带着 [情绪] 说："台词". 背景是 [音效].
```

### 维度三：风格 (Style)

设定视觉和听觉外观：

- **视觉风格**：photorealistic / anime / Chinese ink / film noir
- **光线**：Rembrandt lighting / god rays / neon glow / golden hour
- **色调**：冷色调 / 暖色调 / 高饱和 / 低饱和
- **质感**：film grain / soft blur / sharp / glowing

### 维度四：运镜 (Camera)

独立于场景内容控制摄影机行为：

```
From [角度], [运动] to [景别], using [规则].
```

| 元素 | 示例 |
|------|------|
| 景别 | long shot / medium / close-up / extreme close-up |
| 角度 | eye level / high angle / low angle / POV |
| 运动 | static / zoom in / pan / tilt / crane / tracking |
| 规则 | one-take / no cuts / match cut |
| 速度 | slow / steady / fast / subtle handheld shake |

**核心原则**：用规则而非形容词。不说"电影感"，说"one-take"或"steady follow shot"。

### 维度五：时间线 (Structure)

通过分段控制节奏：

```
0-3s: [事件A]
3-8s: [事件B]
8-12s: [事件C]
12-15s: [结尾方式: freeze frame / camera stop / fade to black]
```

### 维度六：编辑 (Edit)

用于修改已有视频：

| 类型 | 说明 |
|------|------|
| Extend | 在末尾延长 X 秒 |
| Partial Edit | 修改特定元素（发色、服装） |
| Replace | 保留场景，替换主角/产品 |
| Re-Plot | 保留角色/环境，重写故事 |

## 0.4 Markdown 格式化技巧（与 AI 沟通）

Markdown 是与 AI 沟通的最佳格式，比纯文本更结构化，比富文本更通用。

### 关键语法

```markdown
# 一级标题（主题）
## 二级标题（章节）
### 三级标题（子章节）

**加粗**（强调重点）
*斜体*（补充说明）

- 无序列表（并列项）
1. 有序列表（步骤）

> 引用块（重要提示）

`代码`（技术关键词）
​```代码块​```（多行内容）

| 表头 | 表头 |
| --- | --- |
| 内容 | 内容 |
```

### 实战技巧

1. **用标题分层**：让 AI 快速理解文档结构
2. **用列表代替段落**：AI 处理列表比长段落更准确
3. **用代码块包裹 Prompt**：避免 AI 把你的 Prompt 当成指令执行
4. **用表格对比选项**：帮 AI 做选择时效果最好
5. **用引用块标注重点**：`> 注意：这里必须用英文描述`

# 第一章：故事脚本

## 1.1 故事剧本

### 五步种子法（从灵感到剧本）

#### 第一步：种子 (Seed)

种子 = 故事的核心灵感，可以是：
- 一个画面：`一个女孩在暴雨中奔跑`
- 一句话：`如果猫会说话`
- 一种情绪：`毕业季的离别`
- 一个反转：`杀手爱上了目标`

**种子不需要完整，只需要有张力。**

#### 第二步：扩写 (Expand)

用 LLM 根据种子扩写完整剧本。

**Prompt 示例**：

```
你是一个专业的动画编剧。请以「{种子内容}」为核心灵感，
帮我生成一个 {N} 分钟以上的动画剧本。

要求包含：
- 标题
- 总时长
- 风格定位（如：治愈系/热血/悬疑/搞笑）
- 目标受众
- 分场景描述（镜头 + 场景 + 旁白 + 角色行为）
- 每个场景标注情绪走向
```

#### 第三步：角色种子 (Character Seed)

为每个主要角色创建"角色种子卡"：

```
角色名：
年龄：
核心矛盾：（内心冲突，如"渴望自由但被责任束缚"）
标志性动作：（紧张时咬嘴唇、思考时转笔）
说话风格：（简洁/啰嗦/口头禅）
一句话描述：（"一个表面冷漠内心柔软的天才画家"）
```

**角色种子 = 角色一致性的心脏。** 后续所有生图、配音、表演都围绕它展开。

#### 第四步：对话种子 (Dialogue Seed)

好的对话有三层：
1. **字面意思**：角色说了什么
2. **潜台词**：角色真正想说什么
3. **行动目的**：角色说这句话想达到什么效果

**Prompt 示例**：

```
请为以下角色写一段对话场景：
- 角色A：{角色设定}，当前目的：{目的}
- 角色B：{角色设定}，当前目的：{目的}

要求：
- 每句话不超过15个字（口语化）
- 包含潜台词（字面意思 ≠ 真实意图）
- 包含至少一个情绪转折
- 标注每个角色的微表情和肢体语言
```

#### 第五步：剧本格式

标准剧本格式（方便后续分镜）：

```
【场景标题】内景/外景 · 地点 · 时间

（场景描写：环境、光线、氛围）

角色A：（动作/表情）台词内容。

角色B：（动作/表情）台词内容。

（旁白/独白）

【转场提示】切 / 叠化 / 闪黑
```

**后续处理**：
- AI 痕迹重的地方手动改（名字、形容词、冗余描写）
- 删掉无法用画面表达的抽象描写
- 根据目标时长限制字数：2 分钟 ~300-500 字，3 分钟 ~500-800 字，10 分钟 ~2000-3000 字

## 1.2 三幕剧结构（剧本骨架）

三幕剧是最经典的叙事结构，适用于 90% 的故事。

### 结构总览

```
第一幕：铺垫 (Setup)          25%
第二幕：对抗 (Confrontation)  50%
第三幕：结局 (Resolution)     25%
```

### 第一幕：铺垫 (25%)

**目标**：建立世界观、角色、日常生活的平衡

| 节拍 | 说明 | 《海底总动员》示例 |
| --- | --- | --- |
| 开场画面 | 展示主角的日常生活 | Marlin 和 Nemo 在珊瑚礁的家 |
| 日常世界 | 建立角色关系和性格 | Marlin 过度保护，Nemo 渴望独立 |
| **激励事件** | 打破平衡的事件 | Nemo 被潜水员抓走 |
| 拒绝召唤 | 主角最初不愿行动 | Marlin 害怕大海 |
| 跨越门槛 | 主角决定行动 | Marlin 追出珊瑚礁 |

### 第二幕：对抗 (50%)

**目标**：主角面对障碍，逐步成长

| 节拍 | 说明 | 示例 |
| --- | --- | --- |
| 试炼之路 | 遇到盟友和敌人 | Marlin 遇到 Dory、鲨鱼、水母 |
| **中点** | 重大转折（虚假胜利或虚假失败） | 找到线索但 Nemo 在鱼缸里 |
| 反派逼近 | 压力升级，盟友动摇 | Marlin 和 Dory 走散 |
| **最低谷** | 主角失去一切 | Marlin 以为 Nemo 死了 |
| 发现内心力量 | 主角获得关键领悟 | Dory 让 Marlin 相信希望 |

### 第三幕：结局 (25%)

**目标**：最终对决、新平衡

| 节拍 | 说明 | 示例 |
| --- | --- | --- |
| **高潮** | 最终对决/最大考验 | Nemo 在排水管逃脱 |
| 结局 | 展示新平衡 | Nemo 回到大海，Marlin 不再恐惧 |
| 闭合画面 | 与开场形成呼应 | 父子在珊瑚礁，Marlin 微笑送 Nemo 上学 |

### 角色弧线模板

```
角色开始状态：{缺陷/信念}
  ↓ 经历事件
角色认知动摇：{开始质疑}
  ↓ 最低谷
角色被迫面对：{直面缺陷}
  ↓ 领悟
角色转变完成：{新信念/新行为}
```

**示例（Marlin）**：
- 开始：世界很危险，我必须保护 Nemo
- 动摇：追出珊瑚礁后发现大海也有美好
- 最低谷：以为 Nemo 死了，想放弃
- 领悟：过度保护反而伤害了 Nemo
- 转变：学会信任 Nemo，放手让他成长

## 1.3 人物设计

为剧本中每个角色生成详细设定，保证后续人物一致性。

**Prompt 示例**：

```
请帮我生成剧本中每个人物的人物设定，需要具备：
- 服装设定（颜色、材质、风格）
- 外貌特征（发型、发色、体型、年龄）
- 性格特点
- 行为举止 / 习惯性动作
- 情绪状态
- 标志性配饰（如：总是戴着耳机、左手戴银色手链）
```

**关键**：人物设定越详细，后续生图和视频的人物一致性越容易保证。

## 1.4 分镜脚本

基于剧本，用 LLM 生成分镜脚本。**一个分镜对应一个镜头**。

**Prompt 示例**：

```
根据上面的故事剧本，帮我生成分镜脚本。

每个分镜包含：
- 分镜编号
- 镜头类型（远景/全景/中景/近景/特写）
- 镜头运动（推/拉/摇/移/跟/固定）
- 画面描述（场景、人物、动作）
- 对话 / 旁白
- 音效提示
- 预估时长
```

**输出整理**：
- 按 **15 秒为一组** 整理（集梦单段视频上限 15 秒；可灵等平台单段 5-10 秒）
- 编好编号序号（如 `01`、`02`、 `03`...），方便后续剪辑定位
- 连贯的对话/动作尽量放在同一组
- 尽量精简提示词

**每镜头三要素**（来自 visual-skills 最佳实践）：

每个镜头必须包含三个具体细节，缺一不可：
1. **环境压力**：冷冰箱光、潮湿柏油路、闪烁荧光灯
2. **物理微动作**：下巴绷紧、指关节发白、微微颤抖
3. **声音或视觉母题**：钟表滴答、窗外雨滴、反复出现的红色

**禁用词**（不渲染，浪费 token）：`cinematic`, `epic`, `stunning`, `masterpiece`
→ 替换为具体规则：`one-take`, `steady follow shot`, `Rembrandt lighting`

**整理后示例**：

```
【01 | 0-10s】
镜头：近景 → 中景，固定
动作：女主坐在演播台前，放下稿纸，伸懒腰
台词：（无）

【02 | 10-15s】
镜头：近景特写
动作：门被推开，林小小探头进来
台词：冉萱姐！救命！十万火急！

【03 | 15-30s】
镜头：双人近景
动作：林小小冲到女主面前，双手合十
台词：美术系今天请了一位天才漫画家来做分享，原来的主持学长吃坏肚子被拉走了！
```

## 1.5 Prompt 初生成

用 LLM 批量生成两种 prompt，保存到表格中：

**人物形象 prompt**：

```
请把以下角色设定，按照 {目标模型} 的 prompt 格式，
生成每个角色的文生图 prompt。

要求：纯白背景，全身，正面，角色设定如下：
{角色设定内容}
```

**分镜 prompt**：

```
请把以下每个分镜，按照 {目标模型} 的 prompt 格式，
生成对应的文生图 prompt。

分镜脚本如下：
{分镜脚本内容}
```

# 第二章：文生图

这是整个流程中**最重要**的一步，决定了最终画面质量。

## 2.1 画风 / 技术选择

| 方案 | 适用场景 | 说明 |
| --- | --- | --- |
| **现有风格 LoRA** | 最省事 | 在 Civitai 等平台搜索（新海诚、宫崎骏、GPT 手办风、皮克斯 3D 等），下载对应 LoRA + 底模 |
| **自训练 LoRA** | 有喜欢的特定角色/风格 | 截取高清参考图，在 Flux 底模上训练自己的 LoRA |
| **IP-Adapter 风格迁移** | 不推荐做整体风格 | 颜色溢出、干扰主体，目前仅支持 SD1.5/SDXL |

**底模选择**：

| 底模 | 适用 | 说明 |
| --- | --- | --- |
| SD 1.5 | 新手入门 | 丰富、容易调教，配置要求低 |
| SDXL | 商业插图 | 通用商用中坚力量，质量好 |
| Flux | 行业领先 | 高端绘画、影视级效果，吃配置（建议 16G+ 显存） |

**第三方平台**：

| 平台 | 特点 |
| --- | --- |
| 集梦 (Jimeng) | 国内免费，V3 模型效果好，方便 |
| Midjourney | 图像质量高，可用 `--cref` 保持人物一致性 |
| 豆包 | 免费，速度快，质量一般 |
| NoobAI | 清晰度高，抽卡成功率高 |

## 2.2 主要人物形象生成

### 简单做法

生成两张参考图：
1. **面部特写** — 用于 InstantID 保证面部一致
2. **全身形象** — 用于保证服装穿着一致

**局限**：只有一张参考，人物不能太复杂，精细度有限。

### 推荐做法（红标）

用 ControlNet OpenPose 生成：
- **多个面部表情**（喜怒哀惊等）
- **三视图**（正面/侧面/背面）
- **其他骨骼动作**（站、坐、走、跑等）

所有图保持服装外貌一致。生成后**炼制人物 LoRA** — 这是保持人物一致性的最佳方案。

如果没有 LoRA，退而求其次用 InstantID（面部）+ IP-Adapter（服装）。

## 2.3 分镜图生成

### 简单做法（两步走）

**第一步：抽卡出图，确定构图**

- **直接出图**：根据分镜 prompt 不断调试抽卡。时间成本不可控，但能提供构图灵感。
- **参考出图**：找一张喜欢的构图参考（电影截图、Pinterest 等），用图生图 + ControlNet 反推修改。

**第二步：重绘确定细节**（必须做）

构图确定后，用**万物迁移工作流**（ComfyUI）确保人物一致性：

1. 上传参考角色的服装/身体/脸部图片
2. 用遮罩编辑器涂抹需要替换的区域
3. 逐步迁移：先换服装 → 再换脸部 → 调整其他细节
4. 每步调整后点击运行，检查一致性

其他画面调整也用重绘方式。最后确定是否需要**首尾帧**（视频过渡用）。

### 推荐做法

**手绘确定构图**：

1. **线稿构图**：简单画出透视和布局，ControlNet 控制强度不要太高（保留 AI 发挥空间）。需要一定的绘画基础。
2. **简易颜色构图**：用不同颜色色块标示区域（红色=角色 A，蓝色=角色 B，其他=背景），提取颜色通道做遮罩，分区加载 LoRA。不需要透视基础，但需要抽卡。

手绘 → 分区出分镜图 → 延伸首尾帧 → 优化调整。

## 2.4 文生图提示词六大错误（进阶）

### 错误一：用"好看"代替具体描述

- **错误**：`画一个好看的女孩`
- **修正**：`一位18岁东亚女孩，鹅蛋脸，杏仁眼，黑色长发，穿白色校服裙，站在樱花树下`

### 错误二：忽略构图指令

- **错误**：只描述内容，不描述构图
- **修正**：加入构图关键词
  ```
  rule of thirds, subject in center, low angle shot,
  depth of field, bokeh background
  ```

### 错误三：光线描述缺失

- **修正**：光线决定氛围
  ```
  golden hour lighting（黄金时刻）
  rim light（轮廓光）
  dramatic chiaroscuro（明暗对比）
  soft diffused light（柔光）
  neon glow（霓虹光）
  ```

### 错误四：风格混搭冲突

- **错误**：`anime style, photorealistic, oil painting`（风格互相矛盾）
- **修正**：选择一致的风格组合
  ```
  anime style, cel shading, clean lines, vibrant colors
  ```

### 错误五：忽略画面质量词

- **修正**：质量提升关键词
  ```
  masterpiece, best quality, highly detailed,
  8K, UHD, professional photography
  ```

### 错误六：Prompt 过长导致注意力分散

- **修正**：Flux 最佳 prompt 长度 50-150 词
- 核心描述放在前面，修饰词放在后面
- 使用逗号分隔，不要用长句子

## 3.2 视频动态效果（四大常见错误）

### 错误一：缺乏运动动词

- **错误**：`女孩站在风中`（静态描述）
- **修正**：使用**动词优先**思维
  ```
  女孩的长发被风吹起，裙摆随风飘动，
  她微微侧头，眯起眼睛看向远方
  ```

**动词优先原则**：
1. 先想"什么在动" → 头发、裙摆、眼神
2. 再想"怎么动" → 飘起、摆动、转动
3. 最后想"动多快" → 缓慢、自然、剧烈

### 错误二：所有元素都在动

- **错误**：头发在动 + 裙摆在动 + 树叶在动 + 云在动 + 水在动
- **修正**：**选择性运动**，只有 1-2 个元素动，其他静止
  - 主体运动：角色动作（最优先）
  - 环境运动：风吹树叶（辅助）
  - 细节运动：眼神微动（点缀）

### 错误三：运动幅度失控

- **错误**：`角色猛地转身，头发飞舞，衣服猎猎作响`（过度夸张）
- **修正**：控制运动幅度
  - **微动**：眼神、嘴角、手指（细腻情感）
  - **中动**：转头、抬手、走步（日常动作）
  - **大动**：奔跑、跳跃、格斗（动作场景）

### 错误四：忽略画面质量

- **修正**：视频 prompt 也要加质量词
  ```
  cinematic, smooth motion, 24fps, film grain,
  shallow depth of field, natural lighting
  ```

## 3.3 动态运镜（四大常见错误 + 组合镜头）

### 运镜基础术语

| 运镜类型 | 英文 | 说明 | 适用场景 |
| --- | --- | --- | --- |
| 推镜 | Dolly In | 摄影机向前移动 | 强调细节、营造压迫感 |
| 拉镜 | Dolly Out | 摄影机向后移动 | 展示环境、营造孤独感 |
| 变焦推 | Zoom In | 镜头焦距变长 | 聚焦主体、制造紧张 |
| 变焦拉 | Zoom Out | 镜头焦距变短 | 揭示全貌、制造意外 |
| 摇镜 | Pan | 摄影机水平旋转 | 跟随移动、展示场景 |
| 俯仰 | Tilt | 摄影机垂直旋转 | 展示高度、角色登场 |
| 升降 | Crane | 摄影机垂直移动 | 开场/结尾、史诗感 |
| 跟拍 | Tracking Shot | 摄影机跟随主体移动 | 动作场景、沉浸感 |
| 手持 | Handheld | 不稳定手持拍摄 | 紧张、纪实、混乱 |
| 斯坦尼康 | Steadicam | 稳定器跟拍 | 长镜头、流畅移动 |

### 错误一：主体运动 ≠ 摄影机运动

- **错误**：`女孩向前走，镜头向前推`（两者方向相同，画面静止感）
- **修正**：
  - 主体向前走 → 摄影机**固定**或**后退**（产生相对运动）
  - 主体静止 → 摄影机**环绕**或**推近**（创造动态）

### 错误二：运镜节奏不匹配情绪

- **错误**：紧张场景用缓慢运镜，抒情场景用快速运镜
- **修正**：
  - 紧张/追逐 → 快速推拉、手持晃动
  - 浪漫/抒情 → 缓慢推近、平稳移动
  - 震撼/揭示 → 慢速拉远、升降

### 错误三：运镜起点和终点不明确

- **错误**：`镜头移动`（没有说明从哪到哪）
- **修正**：明确起止
  ```
  镜头从角色脚部缓缓上移至面部特写，
  停留在眼部2秒
  ```

### 错误四：忽略运镜转场

- **修正**：用运镜做自然转场
  - 镜头推向角色眼部 → 切到下一场景（主观视角）
  - 镜头快速横摇 → 切到新场景（动作接续）
  - 镜头下降到黑屏 → 时间流逝

### 组合镜头（高级）

组合镜头 = 多种运镜的连续使用，创造电影级效果。

**经典组合**：

```
1. 升降开场（Crane Down）
   → 从天空缓缓下降到街道

2. 跟拍移动（Tracking）
   → 跟随角色穿过人群

3. 推镜聚焦（Dolly In）
   → 推到角色面部特写

4. 摇镜揭示（Pan Reveal）
   → 横摇展示角色看到的景象
```

**组合镜头 Prompt 示例**：

```
开场：镜头从高空缓缓下降，穿过云层，
降落到东京涩谷十字路口（Crane Down）。
跟随一位穿红色连衣裙的女孩穿过人群（Tracking Shot）。
镜头推近到她的面部，她停下脚步，抬头看向天空（Dolly In）。
镜头横摇，揭示她看到的景象：满天樱花（Pan Reveal）。
```

## 3.4 集梦工作流

### 关键设置

| 设置项 | 推荐值 | 说明 |
| --- | --- | --- |
| 模型 | Seens 2.0 Fast | 积分消耗少 |
| 参考模式 | **全能参考** | 保证人物一致性，必须选！ |
| 比例 | 16:9 或 9:16 | 根据发布平台 |
| 时长 | 5-15 秒 | 根据分镜需要 |

### 提示词格式

```
{人物名}@{人物图片}
{场景名}@{场景图片}
{镜头描述}，{动作描述}，{台词}
表情生动，动作富有张力，中文台词，无背景音乐，无字幕
```

- **无字幕**：AI 字幕容易出错，后期剪映自动识别更准
- **无背景音乐**：15 秒一段拼接时 BGM 不连贯，后期统一加

### 声音一致性

从第一段视频提取角色声音，作为后续所有段的音频参考：

1. 生成第一段视频（含台词）
2. 下载 → 在剪映中裁出说话片段（3-5 秒）
3. 分离音频 → 导出为 MP3
4. 后续生成时，在集梦"全能参考"中上传 MP3 作为声音参考
5. 多人对话：每个角色都需要单独提取一段声音参考

### 首尾帧技巧

- **首帧**：镜头开始播放时的第一个画面
- **尾帧**：镜头结束前的最后一个画面
- 中间通过 AI 动画变化实现视觉连贯
- 用于保证镜头之间的过渡自然

**无边画布（Outpainting）**：
- 当首帧和尾帧构图差异大时，用"无边画布"扩展画面边界
- 扩展后的画面作为首尾帧，AI 有更多空间做过渡

### 基础图作为视频源

- 不一定要用"完美"的图做视频源
- 有时候"草图"级别的图 + 好的运动描述 = 更好的视频效果
- 因为 AI 有更多"创作空间"

## 3.5 可灵 AI 工作流

1. 上传分镜图片
2. 描述运动轨迹：**镜头 + 主体 + 运动 + 其他描述**
3. 选择模型（推荐 K2.1）
4. 可开启"生成视频音效"
5. 点击生成

**提示词生成技巧**：把分镜图片上传给 GPT/Gemini，让它根据图片写运镜提示词。

## 3.6 去水印

闭源平台免费版生成的视频有水印，可用 **ProPainter**（GitHub 开源）去除。也可用它移除画面中的物体/角色。

# 第四章：人物一致性（核心难题）

## 4.1 Flux Context（最佳方案）

Flux Context 是目前**最强的角色一致性方案**，通过图片参考 + 文字描述实现精确控制。

### 使用方式

| 方式 | 说明 | 适用 |
| --- | --- | --- |
| 本地部署 | ComfyUI + Flux Context 节点 | 有 16G+ 显存 |
| 云端 API | Replicate / fal.ai 等平台 | 无高配电脑 |
| 在线工具 | 各平台集成版本 | 快速体验 |

### 三步法

**第一步：准备参考图**

- 面部清晰的正面照（光线均匀，无遮挡）
- 全身形象照（展示完整服装）
- 多角度照（侧面、背面，如有）

**第二步：构建 Prompt**

```
角色描述 + 场景描述 + 动作描述 + 风格描述

示例：
Same character as reference image,
standing in a classroom, reading a book,
anime style, cel shading, warm lighting
```

**第三步：迭代调整**

- 面部不一致 → 增加面部参考图权重
- 服装不一致 → 增加全身参考图
- 风格漂移 → 加强风格关键词，降低参考图权重

### Flux Context vs 其他方案

| 方案 | 一致性 | 灵活性 | 易用性 | 成本 |
| --- | --- | --- | --- | --- |
| **Flux Context** | ★★★★★ | ★★★★ | ★★★ | 中 |
| LoRA 训练 | ★★★★★ | ★★★ | ★★ | 高（需训练） |
| InstantID | ★★★★ | ★★★ | ★★★★ | 低 |
| IP-Adapter | ★★★ | ★★★★ | ★★★ | 低 |
| Midjourney --cref | ★★★★ | ★★ | ★★★★★ | 中 |

## 4.2 集梦人物一致性（四种模式）

集梦提供四种人物一致性模式，各有适用场景：

### 模式一：智能参考

- **原理**：AI 自动识别图片中的人物特征
- **适用**：快速出图，不要求精确一致
- **局限**：细节（发型、配饰）可能变化

### 模式二：角色特征

- **原理**：上传角色多角度参考图，AI 提取角色特征
- **适用**：需要较高一致性，但不需要100%精确
- **用法**：
  1. 上传 3-5 张角色参考图（正面、侧面、不同表情）
  2. AI 建立角色特征档案
  3. 后续生成时引用该档案

### 模式三：人像写真

- **原理**：基于真人照片生成一致的角色
- **适用**：需要真人级别的面部一致性
- **局限**：风格受限于写实风格

### 模式四：主体识别

- **原理**：AI 识别图片中的主体（人物/物品/动物）
- **适用**：物品、宠物等非人物的一致性
- **用法**：上传物品图片，后续生成时保持物品外观一致

### 选择建议

| 需求 | 推荐模式 |
| --- | --- |
| 快速出图，大致像就行 | 智能参考 |
| 动漫角色，需要较高一致 | 角色特征 |
| 真人风格，面部必须一致 | 人像写真 |
| 物品/道具一致性 | 主体识别 |

# 第五章：配音与音效

## 5.1 GPT-SoVITS（推荐）

GitHub 开源项目，短时间内语料即可训练语音模型。

# 第六章：剪辑

## 6.1 基本流程

1. 导入所有满意的分镜视频
2. 按编号顺序排列在时间线上
3. 检查语速/节奏：
   - 太快 → 调速 0.8x-0.9x
   - 太慢 → 调速 1.1x-1.2x
4. 添加转场（推荐：叠化、闪黑）
5. 分离音频 → 剪映自动识别字幕 → **人工校对**
6. 添加 BGM 和音效
7. 导出（暂不加字幕，先做画质放大）

## 6.2 剪辑技巧

- 两个镜头衔接不自然 → 加叠化转场
- 语速过快吞字 → 裁开片段，局部调慢
- 一组生成了两个版本 → 各取精华拼接
- 场景突兀切换 → 加闪黑转场营造时间流逝感

# 第七章：视频放大

**为什么放在剪辑之后**：每个视频片段 5 秒 × 20fps = 100 帧。如果剪辑时只用了其中 1-2 秒，前期放大就浪费了。剪辑完成后再统一放大，节省时间和算力。

## 工具：Topaz Video AI

1. 导入剪辑好的视频
2. 选择输出分辨率（2x / 4x / 8x 放大）
3. 帧率补帧（如 20fps → 30fps）
4. 开启：稳定、消除运动模糊、真实差值增强
5. 导出

## 最后：添加字幕

放大后的视频导回剪映，添加字幕，最终导出成片。

## B. 结构化视频 JSON 模板（Video Notation Schema）

> 用 JSON 格式精确控制多场景视频，适合需要高度可控的专业制作。

```json
{
  "metadata": {
    "title": "我的AI短剧",
    "duration_seconds": 180,
    "video_type": "anime_drama",
    "aspect_ratio": "16:9",
    "frame_rate": "24fps",
    "resolution": "1080p",
    "colour_grading": "anime_vibrant"
  },
  "characters": [
    {
      "name": "女主",
      "wardrobe": "白色连衣裙，黑色长发及腰",
      "personality": "温柔但坚强",
      "vocal_traits": "声音清亮，语速适中"
    }
  ],
  "global_style": {
    "camera": "anime style, smooth movements",
    "lighting": "warm golden hour",
    "tone": "治愈系"
  },
  "scenes": [
    {
      "scene_id": "01",
      "location": "教室",
      "time": "下午",
      "subjects": ["女主"],
      "dialogue": "（无）",
      "camera": {
        "shot_size": "medium",
        "movement": "slow dolly in",
        "angle": "eye level"
      },
      "lighting": "soft window light from left",
      "transitions": { "out": "dissolve", "duration": 0.5 }
    }
  ],
  "annotations": {
    "post_production": "需要添加字幕和BGM"
  }
}
```

**核心设计原则**：
- **全局定义**（`global_*`）建立默认值
- **角色档案**（`characters`）定义一次，跨场景复用
- **场景对象**（`scenes`）可覆盖全局设置
- **转场**（`transitions`）明确定义镜头间过渡方式

## C. 连续性守卫（PenShot 架构概念）

> 来自 PenShot (story-shot-agent) 的多 Agent 架构，用于保持跨镜头一致性。

### 三级记忆系统

| 记忆层级 | 保持内容 | 更新频率 |
| --- | --- | --- |
| **短期记忆** | 当前镜头的角色状态、服装、表情 | 每镜头更新 |
| **中期记忆** | 当前场景的环境、光线、道具 | 每场景更新 |
| **长期记忆** | 全局角色档案、风格设定、剧情线 | 全局不变 |

### 连续性检查清单

生成每个新镜头前，检查：
- [ ] 角色服装是否与上一镜头一致？
- [ ] 发型/发色是否变化？
- [ ] 光线方向是否与场景设定一致？
- [ ] 道具位置是否合理？
- [ ] 情绪是否与剧情线匹配？

## E. 积分 / 成本节省技巧

1. **分镜精简**：提示词越短，生成越快
2. **时长估算**：默念台词估算秒数，不填满 15 秒
3. **一次过最好**：抽到满意就不再生成
4. **Fast 模型**：消耗积分最少
5. **复用参考图**：同一场景/人物反复使用
6. **后期放大**：剪辑后再放大，避免放大无用片段
7. **开源替代**：ComfyUI + Wan 2.1 FusionX 可替代部分闭源平台

## F. 常见问题速查表

| 问题 | 解决方案 |
| --- | --- |
| 人物不一致 | 闭源平台用全能参考 + 三视图；ComfyUI 用 LoRA + ControlNet；集梦用角色特征模式 |
| 声音不一致 | 提取第一段音频作为后续声音参考 / GPT-SoVITS 训练 |
| 画面风格冲突 | 统一底模 + LoRA，确保人物和场景风格一致 |
| AI 台词太书面 | 手动改口语化，参考 §1.1 对话种子 |
| 分镜太长/太短 | 按 15 秒分组，估算实际时长 |
| 视频有水印 | 用 ProPainter 去除 |
| 闭源平台太贵 | ComfyUI + Wan 2.1 FusionX 开源替代 |
| 人物情感表达差 | 必须用闭源模型，开源模型目前不适合说话/情感场景 |
| Prompt 太模糊 | 参考 §0.1 六大常见错误 |
| 运镜不自然 | 参考 §3.3 运镜四大错误 |
| 动态效果差 | 参考 §3.2 动态四大错误 |
| 视频过渡生硬 | 使用首尾帧 + 无边画布技巧 |
