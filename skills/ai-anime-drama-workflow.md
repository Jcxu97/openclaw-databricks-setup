# AI 动漫短剧制作全流程 Skill

> 从大纲到成片的完整 AI 动漫短剧制作工作流。

## 工具链

| 环节 | 工具 | 用途 |
| --- | --- | --- |
| 文本生成 | DeepSeek / 其他 LLM | 大纲扩写、脚本生成 |
| 图片生成 | 集梦 (Jimeng) / 豆包 / NoobAI | 人物三视图、场景图、物品图 |
| 视频生成 | 集梦 (Jimeng) | 分镜视频（最长 15 秒/段） |
| 剪辑 | 剪映 (Jianying) | 拼接、字幕、BGM、音效、导出 |

## 全流程概览

```
大纲 → 故事 → 脚本(分镜) → 生图 → 生视频 → 剪辑 → 成片
```

---

## 第一步：大纲

用几句话概括故事核心。示例：

> 女主在演播厅练完播报，被同学拉去临时救场当主持人。到了才发现主讲人是认识的男主。两人互动后产生后续故事。

**要点**：大纲不需要细节，只需明确角色、场景、核心冲突。

---

## 第二步：故事（AI 扩写）

把大纲丢给 LLM，让它扩写成完整故事。

**Prompt 示例**：

```
根据以下大纲，帮我扩写成一个完整的故事：
{大纲内容}
```

**后续处理**：
- AI 痕迹重的地方手动改（名字、形容词、冗余描写）
- 删掉 AI 生成不出来的描写（过于抽象的比喻等）
- 根据目标时长限制字数：
  - 2 分钟 → ~300-500 字
  - 3 分钟 → ~500-800 字
  - 10 分钟 → ~2000-3000 字

---

## 第三步：脚本 / 分镜（AI 生成）

把改好的故事再丢给 LLM，生成分镜脚本。

**Prompt 示例**：

```
根据上面的故事，帮我生成一段 3 分钟动画片的分镜。

分镜格式为：时长 + 镜头 + 动作 + 台词

镜头包括：远景、全景、中景、近景、特写，以及推拉摇移跟。
```

**输出整理**：
- 按 **15 秒为一组** 整理（集梦单段视频上限 15 秒）
- 编好编号序号（如 `01`、`02`、`03`...），方便后续剪辑定位
- 尽量精简提示词，节省集梦积分
- 连贯的对话/动作尽量放在同一组

**整理后示例**：

```
【01 | 0-10s】
镜头：近景 → 中景
动作：女主坐在演播台前，放下稿纸，伸懒腰
台词：（无）

【02 | 10-15s】
镜头：近景特写
动作：门被推开，林小小探头进来
台词：冉萱姐！救命！十万火急！

【03 | 15-30s】
镜头：双人近景
动作：林小小冲到女主面前，双手合十
台词：美术系今天请了一位天才漫画家来做分享，原来的主持学长吃坏肚子被拉走了！
```

---

## 第四步：生图

### 4.1 人物三视图

用 AI 生成角色的 **正面 / 侧面 / 背面** 三视图 + **表情特写**。

**Prompt 要点**：
- 描述形象：发型、发色、服装、体型
- 要求 **纯白背景**，方便后续做参考图
- 平视镜头，双手自然垂下
- 比例 2:3 或 1:1

**生成方式**：
1. 先用 LLM 生成形象描述
2. 在集梦/豆包里生成初版
3. 抽卡直到满意（这是最耗时的环节之一）

**最终需要的素材**：
- 三视图（正面/侧面/背面，同一人物同一服装）
- 表情特写 1-2 张（喜怒哀惊等）
- 不同服装版本（如有换装需求）

### 4.2 场景图

生成故事中出现的场景背景图。

**要点**：
- 风格要和人物一致（不能人物是二次元，场景是写实）
- 人物在场景中不突兀，风格融合
- 可以用 LLM 辅助生成场景描述

### 4.3 物品图

如果故事中有反复出现的道具（如滑板、书本、特定食物），也需要单独生成：
- 三视图
- 45 度角特写（用于画面中）

### 4.4 推荐工具对比

| 工具 | 优点 | 缺点 |
| --- | --- | --- |
| 集梦 (Jimeng) | 方便，直接在平台内用 | 抽卡次数多 |
| 豆包 | 免费，速度快 | 质量一般 |
| NoobAI (NoobAI-XL) | 清晰度高，抽卡成功率高 | 需要额外部署/付费 |

## 第五步：生视频

在集梦中逐段生成视频。

### 5.1 关键设置

| 设置项 | 推荐值 | 说明 |
| --- | --- | --- |
| 模型 | Seens 2.0 Fast | 积分消耗少，速度较慢 |
| 参考模式 | **全能参考** | 保证人物一致性，必须选！ |
| 比例 | 16:9（横屏）或 9:16（竖屏） | 根据发布平台选择 |
| 时长 | 5-15 秒 | 根据分镜需要 |

### 5.2 参考图用法

每段视频需要上传参考图：
- **人物参考**：角色三视图 / 特定服装版本
- **场景参考**：当前段的背景场景图
- **上一帧截图**（可选）：保证镜头衔接连贯

### 5.3 提示词格式

```
{人物名}@{人物图片}
{场景名}@{场景图片}
{镜头描述}，{动作描述}，{台词}
表情生动，动作富有张力，中文台词，无背景音乐，无字幕
```

**说明**：
- **无字幕**：AI 生成的字幕容易出错，后期用剪映自动识别更准
- **无背景音乐**：15 秒一段拼接时 BGM 会不连贯，后期统一加

### 5.4 声音一致性

从第一段视频中提取女主声音，作为后续所有段的音频参考：

1. 生成第一段视频（含台词）
2. 下载第一段视频
3. 在剪映中裁出女主说话的片段（3-5 秒即可）
4. 分离音频 → 导出为 MP3
5. 后续生成时，在集梦的"全能参考"中上传这个 MP3 作为声音参考

**多人对话**：每个角色都需要单独提取一段声音参考 MP3。

### 5.5 文件管理建议

按分镜编号管理文件：

```
项目目录/
├── 第XX集/
│   ├── 01/          # 第一组分镜
│   │   ├── 01-1.mp4  # 第一次生成
│   │   └── 01-2.mp4  # 第二次生成（不满意重做）
│   ├── 02/
│   │   └── 02-1.mp4
│   ├── ...
│   ├── 人物/
│   │   ├── 女主三视图.png
│   │   ├── 女主声音参考.mp3
│   │   ├── 男主三视图.png
│   │   └── 男主声音参考.mp3
│   ├── 场景/
│   │   ├── 教室.png
│   │   └── 舞台.png
│   └── 物品/
│       └── 滑板.png
```

## 第六步：剪辑

在剪映中完成最终剪辑。

### 6.1 基本流程

1. 导入所有满意的分镜视频
2. 按编号顺序排列在时间线上
3. 检查语速 / 节奏：
   - 太快 → 调速 0.8x-0.9x
   - 太慢 → 调速 1.1x-1.2x
4. 添加转场（推荐：叠化、闪黑）
5. 分离音频 → 剪映自动识别字幕 → **人工校对**（AI 字幕必有错字）
6. 添加 BGM 和音效
7. 导出

### 6.2 导出设置

| 参数 | 推荐值 |
| --- | --- |
| 分辨率 | 4K（剪映会员一键超清） |
| 帧率 | 24fps（高帧率会有不自然感） |

### 6.3 常见剪辑技巧

- 两个镜头衔接不自然 → 加叠化转场
- 语速过快吞字 → 裁开片段，局部调慢
- 一组生成了两个版本 → 各取精华拼接（如一个取走姿，一个取侧脸）
- 场景突兀切换 → 加闪黑转场营造时间流逝感

---

## 积分节省技巧

1. **分镜精简**：提示词越短，生成越快
2. **时长估算**：心里默念台词估算秒数，不必要不填满 15 秒
3. **一次过最好**：抽到满意的就不再生成
4. **Fast 模型**：Seens 2.0 Fast 消耗积分最少
5. **复用参考图**：同一场景/人物的参考图可以反复使用

---

## 常见问题

| 问题 | 解决方案 |
| --- | --- |
| 人物不一致 | 必须用"全能参考"模式 + 三视图 |
| 声音不一致 | 提取第一段音频作为后续声音参考 |
| 画面风格冲突 | 确保人物和场景图风格统一 |
| AI 台词太书面 | 手动改口语化 |
| 分镜太长/太短 | 按 15 秒分组，估算实际时长 |
