# AI 动漫短剧制作全流程 Skill

> 从大纲到成片的完整 AI 动漫短剧制作工作流。
> 融合集梦 (Jimeng) 快速工作流与 ComfyUI 专业工作流两种路线。

## 工具链总览

| 环节 | 快速路线（集梦） | 专业路线（ComfyUI + 第三方） |
| --- | --- | --- |
| 文本生成 | DeepSeek / GPT / 豆包 | 同左 |
| 图片生成 | 集梦 / 豆包 / NoobAI | ComfyUI (Flux/SDXL/SD1.5) + ControlNet + LoRA |
| 视频生成 | 集梦（全能参考模式） | 可灵 AI / Runway / 海螺AI / Wan 2.1 FusionX |
| 配音 | 集梦声音参考 | GPT-SoVITS / 音色克隆 / AI 配音网站 |
| 剪辑 | 剪映 | 剪映 / DaVinci Resolve |
| 画质放大 | 剪映会员超清 | Topaz Video AI |

---

## 全流程概览

```
大纲 → 故事剧本 → 人物设计 → 分镜脚本 → Prompt 生成
  → 文生图（人物 + 场景 + 物品）
  → 图生视频
  → 配音 + 配乐
  → 剪辑
  → 视频放大
  → 字幕 → 成片
```

## 第一步：故事脚本（4 个子步骤）

### 1.1 故事剧本

用 LLM 根据主题/灵感扩写完整剧本。

**Prompt 示例**：

```
你是一个专业的动画编剧。请以「{主题}」为主题，
帮我生成一个 {N} 分钟以上的动画剧本。
要求包含：标题、总时长、风格定位、受众、分场景描述（镜头+场景+旁白+角色）。
```

**后续处理**：
- AI 痕迹重的地方手动改（名字、形容词、冗余描写）
- 删掉无法用画面表达的抽象描写
- 根据目标时长限制字数：2 分钟 ~300-500 字，3 分钟 ~500-800 字，10 分钟 ~2000-3000 字

### 1.2 人物设计

为剧本中每个角色生成详细设定，保证后续人物一致性。

**Prompt 示例**：

```
请帮我生成剧本中每个人物的人物设定，需要具备：
- 服装设定（颜色、材质、风格）
- 外貌特征（发型、发色、体型、年龄）
- 性格特点
- 行为举止 / 习惯性动作
- 情绪状态
```

**关键**：人物设定越详细，后续生图和视频的人物一致性越容易保证。

### 1.3 分镜脚本

基于剧本，用 LLM 生成分镜脚本。**一个分镜对应一个镜头**。

**Prompt 示例**：

```
根据上面的故事剧本，帮我生成分镜脚本。

每个分镜包含：
- 分镜编号
- 镜头类型（远景/全景/中景/近景/特写）
- 镜头运动（推/拉/摇/移/跟/固定）
- 画面描述（场景、人物、动作）
- 对话 / 旁白
- 音效提示
- 预估时长
```

**输出整理**：
- 按 **15 秒为一组** 整理（集梦单段视频上限 15 秒；可灵等平台单段 5-10 秒）
- 编好编号序号（如 `01`、`02`、`03`...），方便后续剪辑定位
- 连贯的对话/动作尽量放在同一组
- 尽量精简提示词

**整理后示例**：

```
【01 | 0-10s】
镜头：近景 → 中景，固定
动作：女主坐在演播台前，放下稿纸，伸懒腰
台词：（无）

【02 | 10-15s】
镜头：近景特写
动作：门被推开，林小小探头进来
台词：冉萱姐！救命！十万火急！

【03 | 15-30s】
镜头：双人近景
动作：林小小冲到女主面前，双手合十
台词：美术系今天请了一位天才漫画家来做分享，原来的主持学长吃坏肚子被拉走了！
```

### 1.4 Prompt 初生成

用 LLM 批量生成两种 prompt，保存到表格中：

**人物形象 prompt**：

```
请把以下角色设定，按照 {目标模型} 的 prompt 格式，
生成每个角色的文生图 prompt。

要求：纯白背景，全身，正面，角色设定如下：
{角色设定内容}
```

**分镜 prompt**：

```
请把以下每个分镜，按照 {目标模型} 的 prompt 格式，
生成对应的文生图 prompt。

分镜脚本如下：
{分镜脚本内容}
```

## 第二步：文生图

这是整个流程中**最重要**的一步，决定了最终画面质量。

### 2.1 画风 / 技术选择

| 方案 | 适用场景 | 说明 |
| --- | --- | --- |
| **现有风格 LoRA** | 最省事 | 在 Civitai 等平台搜索（新海诚、宫崎骏、GPT 手办风、皮克斯 3D 等），下载对应 LoRA + 底模 |
| **自训练 LoRA** | 有喜欢的特定角色/风格 | 截取高清参考图，在 Flux 底模上训练自己的 LoRA |
| **IP-Adapter 风格迁移** | 不推荐做整体风格 | 颜色溢出、干扰主体，目前仅支持 SD1.5/SDXL |

**底模选择**：

| 底模 | 适用 | 说明 |
| --- | --- | --- |
| SD 1.5 | 新手入门 | 丰富、容易调教，配置要求低 |
| SDXL | 商业插图 | 通用商用中坚力量，质量好 |
| Flux | 行业领先 | 高端绘画、影视级效果，吃配置（建议 16G+ 显存） |

**第三方平台**：

| 平台 | 特点 |
| --- | --- |
| 集梦 (Jimeng) | 国内免费，V3 模型效果好，方便 |
| Midjourney | 图像质量高，可用 `--cref` 保持人物一致性 |
| 豆包 | 免费，速度快，质量一般 |
| NoobAI | 清晰度高，抽卡成功率高 |

### 2.2 主要人物形象生成

#### 简单做法

生成两张参考图：
1. **面部特写** — 用于 InstantID 保证面部一致
2. **全身形象** — 用于保证服装穿着一致

**局限**：只有一张参考，人物不能太复杂，精细度有限。

#### 推荐做法（红标）

用 ControlNet OpenPose 生成：
- **多个面部表情**（喜怒哀惊等）
- **三视图**（正面/侧面/背面）
- **其他骨骼动作**（站、坐、走、跑等）

所有图保持服装外貌一致。生成后**炼制人物 LoRA** — 这是保持人物一致性的最佳方案。

如果没有 LoRA，退而求其次用 InstantID（面部）+ IP-Adapter（服装）。

### 2.3 分镜图生成

#### 简单做法（两步走）

**第一步：抽卡出图，确定构图**

- **直接出图**：根据分镜 prompt 不断调试抽卡。时间成本不可控，但能提供构图灵感。
- **参考出图**：找一张喜欢的构图参考（电影截图、Pinterest 等），用图生图 + ControlNet 反推修改。

**第二步：重绘确定细节**（必须做）

构图确定后，用**万物迁移工作流**（ComfyUI）确保人物一致性：

1. 上传参考角色的服装/身体/脸部图片
2. 用遮罩编辑器涂抹需要替换的区域
3. 逐步迁移：先换服装 → 再换脸部 → 调整其他细节
4. 每步调整后点击运行，检查一致性

其他画面调整也用重绘方式。最后确定是否需要**首尾帧**（视频过渡用）。

#### 推荐做法

**手绘确定构图**：

1. **线稿构图**：简单画出透视和布局，ControlNet 控制强度不要太高（保留 AI 发挥空间）。需要一定的绘画基础。
2. **简易颜色构图**：用不同颜色色块标示区域（红色=角色 A，蓝色=角色 B，其他=背景），提取颜色通道做遮罩，分区加载 LoRA。不需要透视基础，但需要抽卡。

手绘 → 分区出分镜图 → 延伸首尾帧 → 优化调整。

### 2.4 生图工具对比

| 工具 | 优点 | 缺点 |
| --- | --- | --- |
| 集梦 (Jimeng) | 方便，免费，V3 模型好 | 抽卡次数多 |
| 豆包 | 免费，速度快 | 质量一般 |
| NoobAI (NoobAI-XL) | 清晰度高，抽卡成功率高 | 需要额外部署/付费 |
| ComfyUI + Flux | 可控性最强，质量最高 | 配置要求高，学习曲线陡 |
| Midjourney | 质量高，`--cref` 一致性好 | 付费 |

## 第三步：图生视频

### 3.1 平台对比

#### 闭源平台（推荐，质量高，付费）

| 平台 | 特点 |
| --- | --- |
| **可灵 AI (Kling)** | 人物表情丰富，动作丝滑连贯，一致性好。推荐 K2.1 大模型 |
| **集梦 (Jimeng)** | 全能参考模式保证一致性，支持声音参考 |
| **Runway** | 国外主流，质量高 |
| **海螺 AI** | 国内可用 |

#### 开源项目（免费，吃配置）

| 项目 | 特点 |
| --- | --- |
| **Wan 2.1 FusionX** | 最新，6-10 步出结果，比基础模型快 30-50%，视觉接近闭源。支持图生视频/首尾帧/局部重绘/动作迁移 |
| **Animatediff** | 适合做转场/变化特效，大运动轨迹难 |
| **CogVideo** | 适合无角色的风景动画，人物无情感表达 |

**结论**：需要人物情感/说话 → 必须用闭源模型。纯风景/无角色 → 开源可用。

### 3.2 集梦工作流

#### 关键设置

| 设置项 | 推荐值 | 说明 |
| --- | --- | --- |
| 模型 | Seens 2.0 Fast | 积分消耗少 |
| 参考模式 | **全能参考** | 保证人物一致性，必须选！ |
| 比例 | 16:9 或 9:16 | 根据发布平台 |
| 时长 | 5-15 秒 | 根据分镜需要 |

#### 提示词格式

```
{人物名}@{人物图片}
{场景名}@{场景图片}
{镜头描述}，{动作描述}，{台词}
表情生动，动作富有张力，中文台词，无背景音乐，无字幕
```

- **无字幕**：AI 字幕容易出错，后期剪映自动识别更准
- **无背景音乐**：15 秒一段拼接时 BGM 不连贯，后期统一加

#### 声音一致性

从第一段视频提取角色声音，作为后续所有段的音频参考：

1. 生成第一段视频（含台词）
2. 下载 → 在剪映中裁出说话片段（3-5 秒）
3. 分离音频 → 导出为 MP3
4. 后续生成时，在集梦"全能参考"中上传 MP3 作为声音参考
5. 多人对话：每个角色都需要单独提取一段声音参考

### 3.3 可灵 AI 工作流

1. 上传分镜图片
2. 描述运动轨迹：**镜头 + 主体 + 运动 + 其他描述**
3. 选择模型（推荐 K2.1）
4. 可开启"生成视频音效"
5. 点击生成

**提示词生成技巧**：把分镜图片上传给 GPT，让它根据图片写运镜提示词。

### 3.4 首尾帧

- **首帧**：镜头开始播放时的第一个画面
- **尾帧**：镜头结束前的最后一个画面
- 中间通过 AI 动画变化实现视觉连贯
- 用于保证镜头之间的过渡自然

### 3.5 去水印

闭源平台免费版生成的视频有水印，可用 **ProPainter**（GitHub 开源）去除。也可用它移除画面中的物体/角色。

## 第四步：配音

### 4.1 GPT-SoVITS（推荐）

GitHub 开源项目，短时间内语料即可训练语音模型。

**两种用法**：
1. **下载已训练好的角色模型** — 音色精准，但情感/语速/语调不太可控
2. **音色克隆** — 自己或找人配音 → 音色转换。情感/语速/语调人为可控，但非常吃原本配音者的音色底子

**最佳实践**：配二次元女性时找声音类似的女性配音，配男性时找男性配音。

### 4.2 AI 配音网站

网上有很多文转语音平台，但 AI 味道浓，一听就知道是 AI。适合对配音要求不高的场景。

### 4.3 集梦声音参考

见 §3.2 声音一致性。适合快速路线，不需要额外配音。

## 第六步：剪辑

### 6.1 基本流程

1. 导入所有满意的分镜视频
2. 按编号顺序排列在时间线上
3. 检查语速/节奏：
   - 太快 → 调速 0.8x-0.9x
   - 太慢 → 调速 1.1x-1.2x
4. 添加转场（推荐：叠化、闪黑）
5. 分离音频 → 剪映自动识别字幕 → **人工校对**
6. 添加 BGM 和音效
7. 导出（暂不加字幕，先做画质放大）

### 6.2 剪辑技巧

- 两个镜头衔接不自然 → 加叠化转场
- 语速过快吞字 → 裁开片段，局部调慢
- 一组生成了两个版本 → 各取精华拼接
- 场景突兀切换 → 加闪黑转场营造时间流逝感

---

## 第七步：视频放大

**为什么放在剪辑之后**：每个视频片段 5 秒 × 20fps = 100 帧。如果剪辑时只用了其中 1-2 秒，前期放大就浪费了。剪辑完成后再统一放大，节省时间和算力。

### 工具：Topaz Video AI

1. 导入剪辑好的视频
2. 选择输出分辨率（2x / 4x / 8x 放大）
3. 帧率补帧（如 20fps → 30fps）
4. 开启：稳定、消除运动模糊、真实差值增强
5. 导出

### 最后：添加字幕

放大后的视频导回剪映，添加字幕，最终导出成片。

## 积分 / 成本节省技巧

1. **分镜精简**：提示词越短，生成越快
2. **时长估算**：默念台词估算秒数，不填满 15 秒
3. **一次过最好**：抽到满意就不再生成
4. **Fast 模型**：消耗积分最少
5. **复用参考图**：同一场景/人物反复使用
6. **后期放大**：剪辑后再放大，避免放大无用片段
7. **开源替代**：ComfyUI + Wan 2.1 FusionX 可替代部分闭源平台

## 常见问题

| 问题 | 解决方案 |
| --- | --- |
| 人物不一致 | 闭源平台用全能参考 + 三视图；ComfyUI 用 LoRA + ControlNet |
| 声音不一致 | 提取第一段音频作为后续声音参考 / GPT-SoVITS 训练 |
| 画面风格冲突 | 统一底模 + LoRA，确保人物和场景风格一致 |
| AI 台词太书面 | 手动改口语化 |
| 分镜太长/太短 | 按 15 秒分组，估算实际时长 |
| 视频有水印 | 用 ProPainter 去除 |
| 闭源平台太贵 | ComfyUI + Wan 2.1 FusionX 开源替代 |
| 人物情感表达差 | 必须用闭源模型，开源模型目前不适合说话/情感场景 |
